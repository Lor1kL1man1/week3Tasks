# Week 3 – HTML Laboratory Assignment
**Course:** Multimedia and Web Design  
**Assignment Type:** Individual Practical Exercise  
**Platform:** GitHub Classroom  

---

## 1. Assignment Overview

This laboratory assignment assesses the student’s ability to implement fundamental HTML5 concepts in a structured and technically correct manner.

Students must develop a complete HTML document demonstrating proper use of structural, semantic, and content-related elements.

---

## 2. Project File

All work must be completed inside:

`index.html`

Additional folders (e.g., `images/`) may be created if needed.

---

## 3. Technical Requirements

### 3.1 HTML Document Structure
The page must include:

- `<!DOCTYPE html>`
- `<html>`
- `<head>`
- `<meta charset="UTF-8">`
- `<title>`
- `<body>`

---

### 3.2 Content Elements
Include:

- At least one `<h1>` heading
- At least one additional heading level
- Multiple paragraphs `<p>`
- One horizontal rule `<hr>`

---

### 3.3 Text Formatting
Use the following elements:

- `<strong>`
- `<em>`
- `<mark>`
- `<sup>` or `<sub>`

---

### 3.4 Lists
Create:

- One unordered list `<ul>`
- One ordered list `<ol>`
- One description list `<dl>`

---

### 3.5 Links and Media
Include:

- One external hyperlink
- One hyperlink that opens in a new tab
- One image (local or online)
- A favicon linked inside the `<head>` section

---

### 3.6 Table
Create a structured table that includes:

- `<caption>`
- `<thead>`
- `<tbody>`
- `<tfoot>`
- At least three data rows

---

### 3.7 Form
Create a basic form containing:

- Text input
- Email input
- Password input
- Radio buttons
- Checkbox
- Submit button

---

## 4. Code Quality Standards

Students must:

- Maintain proper indentation
- Close all HTML tags correctly
- Ensure links and images function properly
- Keep code readable and organized

---

## 5. Version Control Requirements

Students must:

- Make at least one commit per exercise section
- Use clear commit messages

Example:

Exercise 1 – HTML Structure
Exercise 2 – Text Formatting
Exercise 3 – Lists


All changes must be pushed before the deadline.

---

## 6. Evaluation Criteria

Assessment will be based on:

- Correct HTML structure
- Proper use of semantic elements
- Code readability
- Completion of all requirements
- Proper use of Git commits

---

## 7. Final Checklist

- [ ] All required elements implemented
- [ ] Code properly formatted
- [ ] Links and images working
- [ ] Required commits completed
- [ ] Repository pushed to GitHub

---

## 8. Academic Integrity

All submitted work must be original and comply with university academic policies.
